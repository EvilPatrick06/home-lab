## 1. Executive Summary

The D&D 5e 2024 character creation system is **substantially implemented** and functional end-to-end. A player can select a class, background, species (with lineage/heritage), assign ability scores via four methods, pick skills, languages, equipment, spells, and save a fully-formed `Character5e` object to disk. The builder supports levels 1–20 including multiclass, level-up, subclass selection at level 3, ASI/feat choices, fighting styles, expertise, epic boons, and higher-level starting equipment.

**Key strengths:**
- All 10 PHB 2024 species with lineages, all 12 base classes, 48 subclasses, 16 backgrounds, 81 feats
- Four ability score methods (standard array, point buy, roll 4d6 drop lowest, custom)
- Equipment A/B/C class choices + background equipment vs 50 GP option
- Versioned character storage with 20-revision history
- Import/export (.dndchar, D&D Beyond JSON, Foundry VTT)
- Auto-save drafts every 30 seconds with 7-day expiry
- Comprehensive 17-rule validation before save

**Key gaps:**
- Artificer class absent (optional/supplemental, not in 2024 PHB)
- Species ability bonuses from 2014-style lineages not wired into stat calculation (only background ASI applied — correct for 2024 rules)
- Builder draft saves only name, game system, and ability scores — not full state
- No enforced step order (free-form navigation, not a wizard)
- Direct URL navigation to edit route without pre-loaded state shows empty builder

## 2. End-to-End Character Creation Flow

### 2.1 Routing

| Route | Component | Purpose |
|-------|-----------|---------|
| `/characters/create` | `CreateCharacterPage` | New character |
| `/characters/5e/create` | `CreateCharacterPage` | New 5e character |
| `/characters/5e/edit/:id` | `CreateCharacterPage` | Edit existing character |
| `/characters/5e/:id` | `CharacterSheet5ePage` | View/interact with sheet |
| `/characters/5e/:id/levelup` | `LevelUp5ePage` | Level-up wizard |

**Source:** `src/renderer/src/App.tsx`

### 2.2 Builder Architecture

The builder is **not a traditional stepper wizard**. It uses a **sidebar + tabbed content area** design:

- **Left sidebar** (`BuildSidebar.tsx`): Shows "build slots" organized by level. Clicking a slot opens a selection modal or custom modal.
- **Right content area** (`MainContentArea5e.tsx`): Five tabs — About, Specials, Languages, Spells, Gear.
- **Modals**: `SelectionModal` (class/species/background/subclass picker), `AbilityScoreModal`, `SkillsModal`, `AsiModal`, `ExpertiseModal`.

### 2.3 Build Slot Sequence

Generated by `generate5eBuildSlots()` in `src/renderer/src/services/character/build-tree-5e.ts` (lines 104–256):

**Foundation (Level 0):**
1. **Class** — selection modal with all 12 classes
2. **Background** — selection modal with 16 backgrounds
3. **Species** — selection modal with 10 species
4. **Ability Scores** — custom modal (standard/point buy/roll/custom)
5. **Skill Proficiencies** — custom modal

**Heritage** — injected dynamically when species has subraces (Dragonborn, Dwarf, Elf, Gnome, Goliath, Halfling, Tiefling).

**Level-based slots (1–20):**
- **Primal Order** (Druid, level 1) — `build-tree-5e.ts` lines 37–41
- **Divine Order** (Cleric, level 1) — lines 43–47
- **Fighting Style** (Fighter L1, Paladin L2, Ranger L2) — lines 30–35
- **ASI** (levels 4, 8, 12, 16; Fighter +6, +14; Rogue +10) — lines 89–102
- **Subclass** (level 3, all classes) — line 26
- **Expertise** (Rogue L1/L6, Bard L2/L9, Ranger L2/L9, Wizard L2) — lines 58–87
- **Epic Boon** (level 19) — lines 214–224

### 2.4 Flow Progression

The `advanceToNextSlot()` action in `build-actions-slice.ts` drives sequential progression:
1. Iterates foundation slots, opens the next unconfirmed slot's modal
2. Then iterates level-up slots in order
3. Special routing: `ability-scores` → opens `customModal: 'ability-scores'`, `skill-choice` → `customModal: 'skills'`, `expertise` → `customModal: 'expertise'`, `ability-boost` → `customModal: 'asi'`

Users **can** click any slot at any time to revisit — the sidebar is fully interactive, not locked.

## 4. Ability Score Generation

### 4.1 Methods Implemented

All four methods are implemented in `ability-score-slice.ts` and rendered in `AbilityScoreModal.tsx`:

| Method | Implementation | Validation |
|--------|---------------|------------|
| **Standard Array** | [15, 14, 13, 12, 10, 8] assigned to abilities via dropdown. Class-suggested arrays available from `ability-score-config.json`. | Must assign all 6 values to confirm (`AbilityScoreModal.tsx` line 52–53) |
| **Point Buy** | Budget: 27 points. Scores 8–15. Cost table: 8→0, 9→1, 10→2, 11→3, 12→4, 13→5, 14→7, 15→9. | Must spend exactly 27 points (`AbilityScoreModal.tsx` line 55–56). UI prevents exceeding budget (`line 79: newTotal <= POINT_BUY_BUDGET`). |
| **Roll (4d6 drop lowest)** | `rollAbilityScores()` in `ability-score-slice.ts` rolls each ability independently. Reroll button available. | Always valid. |
| **Custom** | Manual input 1–20 per ability. | Always valid (no ceiling/floor enforcement in store). |

### 4.2 Ability Score Bonuses

**Background ASI (2024 rules):** 3 points distributed across allowed abilities from background data. Validated in `CharacterBuilder5e.tsx` lines 155–162 — must total exactly 3 points. Applied in `stat-calculator-5e.ts` `calculate5eStats()` line 100.

**Species bonuses:** The 2014-style `abilityScoreIncrease` fields exist in some lineage JSONs but are **not applied** in `calculate5eStats()`. This is **correct for 2024 rules**, where ability score bonuses come from backgrounds, not species.

**ASI (Level-up):** `AsiModal` allows +2 to one ability or +1 to two abilities. Confirmed via `confirmAsi()` which marks the slot and updates scores. Available at levels 4, 8, 12, 16 (all classes), plus Fighter 6/14 and Rogue 10.

### 4.3 Trait-based HP Bonuses

`calculateHPBonusFromTraits()` in `stat-calculator-5e.ts` (lines 72–85):
- Dwarf (Dwarven Toughness): +1 HP per level
- Tough feat: +2 HP per level
- Boon of Fortitude: +40 HP
- Draconic Resilience (Draconic Sorcery, level 3+): +sorcerer level HP

**Verdict:** Fully functional. All four methods work correctly. 2024-style background ASI properly implemented.

## 5. Equipment Selection & Starting Gear

### 5.1 Class Starting Equipment

Each class JSON defines `coreTraits.startingEquipment` as an array of labeled options (A, B, C). The builder renders these as radio buttons in `DetailsTab5e.tsx` (lines 69–250). Selected via `setClassEquipmentChoice()`.

**Build output:** `build-character-5e.ts` lines 160–179 merges:
1. Class equipment from chosen option (with gold from option if applicable)
2. Background equipment OR 50 GP (player choice)
3. Shop-purchased items and trinket

### 5.2 Background Equipment

Two choices rendered in `DetailsTab5e.tsx`:
- **Option A:** Physical equipment list from background JSON
- **Option B:** 50 GP (added to treasure in `build-character-5e.ts` line 580)

Stored as `backgroundEquipmentChoice: 'equipment' | 'gold'`.

### 5.3 Equipment Shop (Gear Tab)

`GearTab5e.tsx` + `EquipmentShop5e.tsx` provide:
- Full searchable equipment catalog (weapons, armor, gear, tools)
- Currency display with editable PP, GP, SP, CP circles
- Buy items (deducts currency with denomination conversion via `deductWithConversion`)
- Item detail views showing damage, type, weight, cost, properties
- Weapon mastery display

### 5.4 Higher-Level Starting Equipment

`HigherLevelEquipment5e.tsx` (for characters created above level 1):
- Gold bonus based on level from `starting-equipment-table.ts` → `starting-equipment.json`
- Magic item slots by rarity (common, uncommon, rare, very rare, legendary)
- Searchable magic item picker per slot
- Roll starting gold option

### 5.5 Trinkets

Trinket roller in `BackstoryEditor5e.tsx` — rolls from `equipment/trinkets.json`. Required for new characters (validation rule 7 in `CharacterBuilder5e.tsx` line 150–153).

### 5.6 Variant Equipment Items

`BackstoryEditor5e.tsx` handles variant items (e.g., "arcane focus" → specific type, "gaming set" → specific set). Validated in rule 6 (lines 136–148).

### 5.7 Currency on Character Sheet

`EquipmentSection5e.tsx` on the character sheet provides:
- Editable PP, GP, EP, SP, CP via `CoinBadge5e`
- Gear shop (search, buy, sell at half price)
- Open packs (e.g., Explorer's Pack → individual items)
- Carrying capacity tracking

**Verdict:** Equipment system is thorough. Class choices, background choices, shop, higher-level gear, trinkets, and variant items all present. The only gap is that the builder draft auto-save doesn't persist equipment choices (only name, system, ability scores).

## 6. Character Sheet Save/Load

### 6.1 Storage Architecture

**Main process storage:** `src/main/storage/character-storage.ts`
- Characters stored as `{userData}/characters/{uuid}.json`
- UUID validation via `isValidUUID()` before save
- Schema versioning: `CURRENT_SCHEMA_VERSION` stamped on every save
- **Versioned backups:** Before overwrite, copies current file to `.versions/{id}/` with ISO timestamp. Keeps latest 20 versions (lines 56–71).
- Migrations: `migrateData()` applies schema v1→v2 (adds `conditions`), v2→v3 (no-op).

### 6.2 IPC Channels

Defined in `src/shared/ipc-channels.ts`:
- `storage:save-character`
- `storage:load-character`
- `storage:load-characters`
- `storage:delete-character`
- `storage:character-versions`
- `storage:character-restore-version`

Handlers in `src/main/ipc/storage-handlers.ts` (lines 57–91).

### 6.3 Renderer Save Flow

1. Builder validation passes (17 rules, `CharacterBuilder5e.tsx` lines 100–274)
2. `buildCharacter5e()` called — assembles full `Character5e` object from store state (665-line function in `build-character-5e.ts`)
3. `useCharacterStore.saveCharacter()` → `window.api.saveCharacter()` → IPC → main process `saveCharacter()`
4. Navigate to character sheet

### 6.4 Load/Edit Flow

1. `loadCharacterForEdit(character)` in `save-slice.ts` delegates to `loadCharacterForEdit5e()` in `load-character-5e.ts`
2. Regenerates build slots from `generate5eBuildSlots(character.level, classId)`
3. Restores all slot selections: species, class, background, ability scores, skills, subclass, epic boon, primal/divine order, fighting style, ASI, expertise
4. Restores character details: name, icon, alignment, appearance, personality, equipment, currencies, languages, spells
5. Async loads SRD data to restore heritage slots, class equipment, background equipment, skills data

### 6.5 Import/Export

**File:** `src/renderer/src/services/io/character-io.ts`
- `exportCharacterToFile()`: Native save dialog → `.dndchar` file (JSON)
- `importCharacterFromFile()`: Native open dialog → parse + validate (id, gameSystem, name)
- **Additional imports** (via `ViewCharactersPage.tsx`): D&D Beyond JSON, Foundry VTT actor JSON

### 6.6 Auto-Save Drafts

**File:** `src/renderer/src/hooks/use-auto-save.ts`
- Saves to `localStorage` every 30 seconds (`DRAFT_SAVE_INTERVAL = 30_000`)
- Stores only: `characterName`, `gameSystem`, `abilityScores`, `timestamp`
- Drafts expire after 7 days
- `CreateCharacterPage` shows resume modal if draft exists

### 6.7 `buildChoices` Persistence

The `Character5e.buildChoices` object preserves all builder state for re-editing:

```
speciesId, subspeciesId, classId, subclassId, backgroundId,
selectedSkills, abilityScoreMethod, abilityScoreAssignments,
asiChoices, chosenLanguages, backgroundAbilityBonuses,
versatileFeatId, epicBoonId, generalFeatChoices, fightingStyleId,
backgroundEquipmentChoice, classEquipmentChoice,
primalOrderChoice, divineOrderChoice, elementalFuryChoice,
speciesSpellcastingAbility, keenSensesSkill,
blessedWarriorCantrips, druidicWarriorCantrips,
expertiseChoices, multiclassEntries
```

**Verdict:** Save/load is robust. Versioned backups, full `buildChoices` round-tripping, import/export with multiple format support. The draft auto-save is minimal (only 3 fields) — a full-state draft would be more resilient.

### 7.3 Validation (17 Rules)

All validated in `CharacterBuilder5e.tsx` lines 100–274:

1. Character name required
2. Foundation slots (species, class, background, ability scores, skills) all filled
3. Alignment required
4. Background equipment choice required
5. Class equipment choice required
6. Equipment variant choices resolved
7. Trinket required (new characters only)
8. Background ability bonuses total 3 points
9. Skill proficiencies complete
10. Human Versatile feat required
11. Species size choice required (Human, Tiefling)
12. Heritage required (if species has subraces)
13. Species spellcasting ability required (Elf, Tiefling, Gnome)
14. Keen Senses skill required (Elf)
15. Languages complete
16. Class spell selection complete (cantrips + prepared)
17. Fighting style cantrip validation (Blessed Warrior, Druidic Warrior)

Save button disabled until all 17 rules pass. First validation issue shown in UI.

### 7.4 UX Strengths

- **Live summary bar** showing ability mods, saves, AC, speed updates in real-time
- **Free navigation** — users can revisit any slot or tab at any time
- **Selection modal** with search, rarity filter, and preview pane
- **Class-suggested standard arrays** for ability scores
- **Incomplete detail detection** — modal warns about blank optional fields before save
- **Draft persistence** — resume interrupted creation sessions

### 7.5 UX Weaknesses

- **No guided flow** — new D&D players may not know which order to fill things in
- **Specials tab dependency** — requires background selection first; shows message but no redirect
- **No progress indicator** — no percentage or step counter showing how much is complete
- **Edit via URL unreliable** — `/characters/5e/edit/:id` doesn't load from `:id` param; requires pre-loading through "Re-Make Character" button on sheet

## 8. Gaps, Bugs, and Partial Implementations

### 8.1 Missing Features

| Gap | Severity | Details |
|-----|----------|---------|
| **Artificer class** | Low | Not in 2024 PHB. Would require new data + build-tree logic. |
| **Full draft auto-save** | Medium | Only saves name, system, ability scores. Equipment, spells, skills, background choices all lost on crash. |
| **Progress indicator** | Low | No step counter or completion percentage in the wizard. |
| **Guided mode for new players** | Medium | No optional "guided" mode that locks progression and explains each step. |
| **Multiclass spellcasting table** | Low | `computeSpellcastingInfo` may not fully combine spell slot tables for multiclass casters. |

### 8.2 Architectural Issues

| Issue | File | Details |
|-------|------|---------|
| **`phase: 'complete'` never set** | `stores/builder/types.ts` | `BuilderPhase` includes `'complete'` but nothing transitions to it; builder stays in `'building'` post-save. |
| **`customModal` in wrong slice** | `character-details-slice.ts` | `customModal` is UI state but lives in the character details slice rather than a dedicated UI slice. |
| **Async race in `setClassEquipmentChoice`** | `character-details-slice.ts` lines 110–132 | `load5eClasses()` resolves in `.then()` after setState; if store changes first, could overwrite with stale data. |
| **`addEquipmentItem` always targets classEquipment** | `character-details-slice.ts` | No `addEquipmentItem` path for `bgEquipment` — only `removeEquipmentItem` checks both. |
| **Edit URL doesn't self-hydrate** | `CreateCharacterPage.tsx` | Route `/characters/5e/edit/:id` doesn't use `useParams` to load character; requires `loadCharacterForEdit` called before navigation. Direct URL visit or page refresh shows empty builder. |

### 8.3 Store Validation Gaps

| Gap | Details |
|-----|---------|
| **Point buy budget not enforced in store** | Only UI (`AbilityScoreModal`) prevents overspending; store accepts any scores. |
| **Skill count not enforced in store** | `selectedSkills` can exceed `maxSkills` if set programmatically. |
| **Spell count not enforced in store** | Selected spells not validated against cantrip/prepared maximums in store. |

### 8.4 Data Gaps

| Data | Status | Details |
|------|--------|---------|
| Spells | ~400 present | Full 2024 PHB list may be slightly larger; hard to audit without official count. |
| Magic items | Present by rarity | Organized in `equipment/magic-items/` by rarity tier. |
| Class features | Present | `character/class-features/` with per-class JSON. Loaded via `load5eClassFeatures()`. |
| Weapon masteries | Present | Weapons include `mastery` property (Vex, Nick, Push, Topple, etc.) |
