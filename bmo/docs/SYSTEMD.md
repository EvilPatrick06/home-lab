# BMO Systemd Services

5 systemd services manage BMO's runtime. Each has a `.service` file in `/etc/systemd/system/`.

## Services overview

| Service | Owner file (in repo) | What it runs |
|---|---|---|
| `bmo.service` | generated by `bmo/setup-bmo.sh` | `/home/patrick/home-lab/bmo/pi/venv/bin/python app.py` |
| `bmo-fan.service` | generated by `bmo/setup-bmo.sh` | `/usr/bin/python3 /home/patrick/home-lab/bmo/pi/hardware/fan_control.py` |
| `bmo-kiosk.service` | `bmo/pi/kiosk/bmo-kiosk.service` | Chromium fullscreen kiosk on HDMI |
| `bmo-dm-bot.service` | `bmo/pi/kiosk/bmo-dm-bot.service` | `python -m bots.discord_dm_bot` |
| `bmo-social-bot.service` | `bmo/pi/kiosk/bmo-social-bot.service` | `python -m bots.discord_social_bot` |

Optional:

| Service | Owner file | What it runs |
|---|---|---|
| `bmo-ide.service` | `bmo/pi/ide_app/bmo-ide.service` | Embedded web IDE on port 5001 |
| `bmo-backup.service` + `.timer` | *recoverable from git history (`git log --all --full-history -- bmo/docker/bmo-backup.service`)* | Daily backup of `data/` — legacy, not installed |

## Service dependencies

```
bmo.service                    (main Flask — starts first)
  └─► bmo-kiosk.service        (needs /health endpoint, ExecStartPre polls it)
  └─► bmo-dm-bot.service       (talks to main BMO on :5000)
  └─► bmo-social-bot.service   (talks to main BMO on :5000)

bmo-fan.service                (independent — just I2C to fan hat)
```

## Common commands

```bash
# Status
systemctl status bmo                                       # one service
systemctl status bmo bmo-fan bmo-kiosk bmo-dm-bot bmo-social-bot    # all
systemctl list-units --type=service --state=active | grep bmo

# Control
sudo systemctl start bmo
sudo systemctl stop bmo
sudo systemctl restart bmo
sudo systemctl reload bmo                  # (not implemented currently)

# Enable at boot
sudo systemctl enable bmo
sudo systemctl disable bmo

# After changing a .service file
sudo systemctl daemon-reload               # pick up new config
sudo systemctl restart bmo

# Install or update a service file from repo
sudo cp ~/home-lab/bmo/pi/kiosk/bmo-kiosk.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now bmo-kiosk
```

## Log viewing

```bash

journalctl -u bmo -n 100 --no-pager
journalctl -u bmo --since "10 min ago" --no-pager
journalctl -u bmo --since "2026-04-23 15:00" --until "2026-04-23 16:00"

# Errors only
journalctl -u bmo -p err --no-pager

# Since boot
journalctl -u bmo -b

# All BMO services merged
journalctl -u bmo -u bmo-kiosk -u bmo-dm-bot -u bmo-social-bot --since "1 hour ago"
```

## Service file locations

Installed (live): `/etc/systemd/system/bmo*.service`
Source in repo: `bmo/pi/kiosk/bmo-*.service` + generated by `bmo/setup-bmo.sh`

**If you edit a .service file in the repo, the installed copy is NOT auto-updated.** Manual step:

```bash
# Re-install from repo
sudo cp ~/home-lab/bmo/pi/kiosk/bmo-kiosk.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl restart bmo-kiosk
```

OR re-run the bootstrap script:

```bash
cd ~/home-lab/bmo
bash setup-bmo.sh        # idempotent — rewrites service files
sudo systemctl daemon-reload
sudo systemctl restart bmo bmo-fan bmo-kiosk bmo-dm-bot bmo-social-bot
```

## Paths in service files (canonical)

All paths are absolute:

```ini
# bmo.service (main Flask)
[Service]
User=patrick
Group=patrick
WorkingDirectory=/home/patrick/home-lab/bmo/pi
Environment=PATH=/home/patrick/home-lab/bmo/pi/venv/bin:/usr/local/bin:/usr/bin:/bin
EnvironmentFile=/home/patrick/home-lab/bmo/pi/.env
ExecStart=/home/patrick/home-lab/bmo/pi/venv/bin/python app.py
Restart=on-failure
RestartSec=5
```

```ini
# bmo-dm-bot.service (module-style launch, needed since bots/ subpackage)
[Service]
User=patrick
Group=patrick
WorkingDirectory=/home/patrick/home-lab/bmo/pi
EnvironmentFile=/home/patrick/home-lab/bmo/pi/.env
ExecStart=/home/patrick/home-lab/bmo/pi/venv/bin/python -m bots.discord_dm_bot
Restart=on-failure
```

```ini
# bmo-fan.service (uses system Python for smbus — NOT venv)
[Service]
User=patrick
Group=patrick
ExecStart=/usr/bin/python3 /home/patrick/home-lab/bmo/pi/hardware/fan_control.py
Restart=on-failure
RestartSec=10
```

**Do NOT** use relative paths in service files. systemd doesn't resolve them.

## Restart policies

All BMO services use:

```ini
Restart=on-failure
RestartSec=5                    # for most
RestartSec=10                   # for bmo-fan (hardware init slower)
```

This means:
- Normal exit (code 0) → no restart
- Crash (signal, non-zero exit) → restart after 5-10 seconds
- Manual `systemctl stop` → stays stopped

Rate limit (via `StartLimitIntervalSec=300` + `StartLimitBurst=5`): if BMO crashes 5x in 5 min, systemd gives up. Fix or manually restart:

```bash
sudo systemctl reset-failed bmo
sudo systemctl start bmo
```

## Security hardening (optional)

Harden service sandboxing:

```ini
[Service]
# Filesystem
ProtectSystem=strict                   # /usr, /boot, /etc read-only (needs paths whitelisted)
ReadWritePaths=/home/patrick/home-lab/bmo/pi/data /home/patrick/home-lab/bmo/pi/config /tmp
ProtectHome=read-only                  # unless writing to own home
NoNewPrivileges=true                   # no setuid escape
PrivateTmp=true                        # isolated /tmp

# Kernel
ProtectKernelTunables=true
ProtectKernelModules=true
ProtectKernelLogs=true
ProtectControlGroups=true

# Network (restricted allowlist if needed)
# RestrictAddressFamilies=AF_INET AF_INET6 AF_UNIX
```

Currently BMO uses minimal hardening. Add as comfort increases.

## Troubleshooting service issues

See [`TROUBLESHOOTING.md`](./TROUBLESHOOTING.md).

Quick diagnostics:

```bash
# Service failed to start
systemctl status bmo
journalctl -u bmo -n 50 --no-pager

# Service keeps crashing
systemctl show bmo --property=NRestarts
# high number = crash loop

# Service active but endpoints hang
ss -tnp | grep 5000
curl --max-time 3 -v http://localhost:5000/health
```

## Adding a new systemd service

1. Write `.service` file in `bmo/pi/kiosk/` (or wherever makes sense)
2. Install:
   ```bash
   sudo cp bmo-my-service.service /etc/systemd/system/
   sudo systemctl daemon-reload
   sudo systemctl enable --now bmo-my-service
   ```
3. Verify: `systemctl status bmo-my-service`
4. Update `bmo/setup-bmo.sh` to install it on fresh setups
5. Document in this file

## Disabling services (temporarily)

If BMO is crashing and you need to stop the crash loop:

```bash
sudo systemctl stop bmo
sudo systemctl disable bmo       # prevents auto-start at boot

# fix the bug...

sudo systemctl enable bmo
sudo systemctl start bmo
```
