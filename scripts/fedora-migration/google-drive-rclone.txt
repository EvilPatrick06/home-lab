Google Drive folder for backups (Windows11BackUp)
==================================================

Automated setup on Windows (browser = Google Allow only, after you add OAuth client id+secret):
  1) Copy gdrive-oauth.local.env.example -> gdrive-oauth.local.env  (gitignored)
  2) Paste Desktop OAuth client id + secret from Google Cloud Console
  3) Run:  .\complete-gdrive-rclone-setup.ps1
     That creates remote "gwin" + root_folder_id below, then runs reconnect (opens default browser for consent).

Optional: one tab to create credentials —  .\open-drive-oauth-on-your-screen.ps1

If you see "OAuth client was not found" or "Error 401: invalid_client"
------------------------------------------------------------------------
  Google is rejecting the OAuth app. Fix: use YOUR OWN client ID (do not leave blank).

  Full official steps: https://rclone.org/drive/#making-your-own-client-id

  Short version:
  1) https://console.cloud.google.com/ — pick or create a project.
  2) APIs & Services → Enable APIs → enable "Google Drive API".
  3) OAuth consent screen: External, add yourself as a Test user; Data Access: add scopes
     https://www.googleapis.com/auth/drive (and drive.metadata.readonly / docs if you want full rclone).
  4) APIs & Services → Credentials → Create Credentials → OAuth client ID →
     Application type: **Desktop app** (not "Web"). Create; copy Client ID and Client secret.
  5) Run rclone config again. When asked for client_id and client_secret, **paste those values**
     (do not press Enter to skip).
  6) Use auto config; browser should open to http://127.0.0.1:53682/ — allow if firewall asks.

  If you get redirect_uri_mismatch: open that OAuth client in Google Console → add Authorized redirect URI:
     http://127.0.0.1:53682/
  (Desktop app type usually pre-includes loopback URIs; Web app type causes this error.)

rclone: make this folder the root of a dedicated remote (simplest — paths are just filenames)
------------------------------------------------------------------------------------------------
  1) rclone config
  2) New remote, type drive, name e.g. gwin
  3) When prompted, enter your **own** client_id and client_secret (see above).
  4) When asked for "root_folder_id", paste:
       1cntyEzO2tdhQwAr8kU9xy_G6DHTmHoPu
  5) Save.

Then uploads go straight into Windows11BackUp:
  rclone copy backup.tar.zst gwin: --progress

Alternative: keep default My Drive remote and use a path (if that folder lives under My Drive):
  rclone copy backup.tar.zst gdrive:Windows11BackUp/ --progress

Match the remote name to your backup scripts:
  RCLONE_REMOTE=gwin ./backup-to-gdrive.sh
